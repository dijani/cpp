Funcții

Orice problema poate fi divizată în subprobleme mici. Astfel orice program poate fi compus din mai multe subprograme. Subprogram reprezintă un fragment al programului (program principal). Utilizarea subprogramelor face textul programului mai scurt și mai ușor de conceput. Un avantaj al subprogramelor constă în faptul că o dată definit el poate fi utilizat de număr dorit de ori (apel de subprogram). În limbajul C++ subprogramele sunt reprezentate sub forma de funcții.

Cu o funcție main noi deja suntem cunoscuți. Ea reprezintă blocul principal al programului în C++. În afară de aceasta noi putem defini funcții proprii. Orice funcție trebuie definită înainte de blocul funcției main (există posibilitatea de definire și mai jos de blocul main), cu o structură asemănătoarea cu main: antetul funcției(prima linie) și corpul ei.

Sintaxa declarației de funcție este:

  • tip_rezultat nume_functie(parametrii_formali) {
  • corpul funcției
  • }
Exemplu:
  • int suma (int a, int b) {
  • return a+b;
  • }

  • se definește o funcție cu numele suma cu 2 parametri formali întregi a și b (argumentele funcției), rezultatul funcției este de tip întreg
  • în corpul funcției se realizează operația de adunare și rezultatul se întoarce în main prin operatorul return
După ce funcția este definită, în blocul main ea trebuie chemată pentru execuția (apelarea funcției). Funcția se apelează utilizând numele ei și parametrii actuali (valorile concrete ce se transmit la funcția).
Exemplu:
  • int suma (int a, int b) {
  • return a+b;
  • }
  • int main() {
  • int x,y;
  • cout<<"Introdu 2 valori intregi:"<<endl;
  • cin>>x>>y;
  • cout<<"Suma acestor numere este:"<<suma(x,y);
  • return 0;
  • }

  • după definirea funcției înainte de blocul main, ea este chemată în operatorul cout
  • apelul este compus din numele funcției suma și 2 parametri actuali x și y
Remarcă
Funcțiile des sunt utilizate pentru realizarea calculelor și obținerea unui rezultat ce trebuie întors în blocul main.
Însă este posibil să definim o funcție ce nu realizează calcule și nu întoarce nici un rezultat în blocul main. Astfel de funcții la tipul rezultatului au tipul void.
Exemplu:
  • void salutare (string s) {
  • cout<<"Salut, " + s;
  • }

  • se definește o funcție cu numele salutare cu 1 parametru formal de tip șir de caractre s (argumentul funcției), care nu returnează nimic (void)
  • în corpul funcției se utilizează operatorul cout ce afișează rezultatul concatenării a 2 șiruri: Salut, și valoarea argumentului s
Apelul astfel de funcții se diferă de apelul funcției cu rezultat.
Exemplu:
  • void salutare (string s) {
  • cout<<"Salut, " + s;
  • }
  • int main() {
  • salutare("Maria");
  • return 0;
  • }

  • după definirea funcției înainte de blocul main, funcția salutare este chemată cu parametrul actual Maria ca instrucțiunea independentă
Remarcă
Un material util despre funcții puteți găsi pe următorul link.

În continuare este prezentat un exemplu de program cu câteva funcții.
  • #include <iostream>
  • using namespace std;
  • bool min(int a, int b){
  • return a < b;
  • }
  • void salutare() {
  • cout<<"Salut, prietene!"<<endl;
  • }
  • int main()
  • {

Funcții recursive

Pentru înțelegerea mai bună a recursiei este nevoie să facem cunoștință cu vizibilitatea declarației și noțiunile de variabila locala vs variabila globală.
Variabila locală se declară și se utilizează în interiorul blocului unei funcții. Pe când variabila globală se declară în afară blocului unei funcții, dar poate fi utilizată în toate blocurile de funcții descrise mai jos. Astfel variabila locală este vizibilă deaceea poate fi utilizată numai în blocul unde este declarată. Dar variabila globală este vizibilă peste tot ceea ce permite utilizarea ei în alte blocurile ale funcțiilor.
Remarcă
Deoarece variabila globală este vizibilă peste tot și poate fi utilizată în blocurile altor funcții, înseamnă că în blocurile respective valoarea variabilei globale poate fi modificată. Acest lucru nu întotdeauna este benefic, efectul negativ al acestei modificări se numește efectul colateral(secundar).
Recomandarea: Comunicarea cu subprograme la nivel de argumente realizați utilizând parametrii, ceea ce permite modularea programului sau utilizarea lui universală.

La definirea funcțiilor numele ei fiind declarația cu tip void sau tip specific rezultatului este vizibilă în blocul funcției respective, ceea ce permite apelarea funcției date nu numai în blocul altei funcții dar și în blocul curent. Aceasta situație și se numește recursia: apelarea funcției în interiorul blocul său sau autoapelarea funcției.
Conceptul recursivității poate fi explicat astfel: Recursia este proprietatea unui obiect de a se defini prin el însuși. Conceptul dat ușor se înțelege vizual. Pentru vizualizarea recursivității să privim obiecte de artă.

Pentru elaborarea unei funcții recursive este nevoie de exprimat recursia prin 2 cazuri: cazul elementar și cazul de recurență. Cazul elementar reprezintă o valoare cunoscută ce indică condiția de terminare a procesului de recursie. Cazul de recurență reprezintă autoapelul care neapărat trebuie să rezulte spre cazul elementar, adică spre valoarea inițială (condiția de terminare a recursiei). Aceste 2 cazuri sunt definite într-o instrucțiunea condițională IF.
Exemplu cu funcția recursivă de tip void:
Afișarea numerelor în descreștere.
  • void afis (int n) {
  • if (n == 0) return;
  • else {
  • cout<<n<<' ';
  • afis(n-1);
  • }
  • }
  • int main() {
  • afis(5);
  • return 0;
  • }

  • În rezultatul execuției programului dat se vor afișa numerele: 5 4 3 2 1
Exemplu cu funcția recursivă de tip int:
Calculul sumei cifrelor dintr-un număr.
  • int sumcifre (int n) {
  • if (n == 0) return 0;
  • else return sumcifre(n / 10) + n % 10;
  • }
  • int main() {
  • cout<<"Suma cifrelor = "<<sumcifre(123);
  • return 0;
  • }

  • În rezultatul execuției programului dat se va afișa: Suma cifrelor = 6
La programarea recursivă se cunoaște recursia directă și indirectă. Ceea ce a fost prezentat mai sus reprezintă recursii directe. Recursia indirectă presupune autoapelarea funcțiilor prin intermediul altor funcții.
Sintaxa schematică recursiei indirecte este:
  • tip b(parametrii_formali) {
  • ...
  • a(parametrii_actuali); // funcția b apelează funcția a
  • ...
  • }
  • tip a(parametrii_formali) { // funcția a este recursivă, apelează funcția b care apelează pe funcția a
  • ...
  • if (conditie)
  • ...
  • b(parametrii_actuali);
  • ...
  • }
